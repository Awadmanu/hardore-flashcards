# Flashcard Reviewer

A desktop CSV flashcard review and editing tool built with Python and Tkinter. The app lets you load flashcards from a CSV file, review questions one by one, reveal or hide answers, edit cards, insert new cards, delete cards, save changes, and attach images to flashcard questions or answers.

## Introduction

Flashcard Reviewer is a lightweight study tool for reviewing and maintaining flashcards stored in CSV format. It is designed for simple workflows where flashcards can be opened in spreadsheet tools, generated by external tools, or edited directly inside the app.

The main application uses a dark Tkinter interface and supports a four-column CSV structure:

```csv
Question,Answer,Difficulty,Topic
```

## Features

- Load flashcards from a `.csv` file.
- Review cards one at a time.
- Show or hide answers during study.
- Optionally show answers automatically.
- Navigate with buttons or keyboard shortcuts.
- Add cards before or after the current card.
- Edit the current card.
- Delete cards.
- Save changes back to CSV.
- Track unsaved changes.
- Use difficulty labels such as `Basic`, `Intermediate`, and `Difficult`.
- Group cards by topic.
- Paste images into questions or answers.
- Store pasted images in a media folder next to the CSV file.
- Render image tags inside flashcards.
- Includes helper scripts for Windows builds and icon generation.

## CSV Format

The expected CSV header is:

```csv
Question,Answer,Difficulty,Topic
```

Each row represents one flashcard:

```csv
"What is photosynthesis?","The process by which plants convert light energy into chemical energy.","Basic","Biology"
```

### Columns

| Column       | Description                                                          |
| ------------ | -------------------------------------------------------------------- |
| `Question`   | The prompt shown on the front of the card.                           |
| `Answer`     | The answer revealed during review.                                   |
| `Difficulty` | A difficulty label, such as `Basic`, `Intermediate`, or `Difficult`. |
| `Topic`      | A subject or category label used to organize cards.                  |

## Usage

Run the main application:

```bash
python claude_image_flashcards.py
```

Then:

1. Click **Cargar CSV** to open a flashcard CSV file.
2. Review the current question.
3. Press **Space** or click **Ver respuesta** to reveal the answer.
4. Use **Anterior** and **Siguiente** to move between cards.
5. Use **Editar**, **＋ Antes**, **＋ Después**, or **Eliminar** to manage cards.
6. Click **Guardar CSV** to save changes.

## Image Support

The image-enabled version supports embedded image references using this syntax:

```text
[[img:filename.png]]
```

When an image is pasted into the editor, it is saved as a PNG file in a media directory next to the CSV file. For example, if the CSV is:

```text
my_flashcards.csv
```

the media folder will be:

```text
my_flashcards_media/
```

Images can be inserted into either the `Question` or `Answer` fields.

> Note: Image paste support requires Pillow.

## Build for Windows

A Windows build script is included:

```bash
build_windows.bat
```

The script checks for Python, installs build dependencies, looks for `claude_image_flashcards.py`, optionally uses `icon.ico`, and builds a single-file `.exe` with PyInstaller.

The generated executable is placed in:

```text
dist/Flashcard Reviewer.exe
```

To generate a Windows `.ico` file from an image:

```bash
python make_icon.py original_image_GPT.png -o icon.ico
```

## Troubleshooting

### The app cannot open my CSV

Make sure the CSV contains the required header:

```csv
Question,Answer,Difficulty,Topic
```

Also confirm the file is encoded in UTF-8 or a compatible encoding.

### Images do not appear

Check that:

- Pillow is installed.
- The image file exists in the correct `{csv_name}_media/` folder.
- The CSV contains a valid image tag such as `[[img:example.png]]`.

### Pasting images does not work

Image paste support depends on clipboard image support in the operating system and on Pillow being installed.

Install Pillow with:

```bash
pip install pillow
```

### The Windows build fails

Try these steps:

```bash
pip install --upgrade pip
pip install pyinstaller pillow
build_windows.bat
```

Make sure `claude_image_flashcards.py` is in the same folder as `build_windows.bat`.
